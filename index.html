STANDARDIZED PROCESS

Every SAGE Action Follows Four Steps

A standardized system prompt ensures consistency, traceability, and quality across every interaction. The human always approves the final MCP call.

STEP 01

Pull Metadata

Retrieve work item context from DevOps/Cosmos DB — title, discussions, metadata, and full project history.

STEP 02

Reference Best Practices

Query Microsoft Learn via MCP for the specific action's best practices, patterns, and documentation.

STEP 03

Clarify Context

Ask three targeted clarifying questions using RAG against the full Cosmos DB project database.

STEP 04

Execute Action

Combine best practices with the specific action, present options to the user, and execute upon approval via MCP.

ARCHITECTURAL DECISIONS

Key Decisions Made

Abandon 80+ bespoke agents

Build SAGE as a centralized, generic agent front-end with curated prompts mapped to RSM Velocity actions.

DevOps remains source of truth

Associates' day-to-day workflows do not change. SAGE reads from and acts upon DevOps data.

Cosmos DB over direct DevOps API

All work items synced to Cosmos DB for performance. Delta sync on every user login.

SAGE as button in Engagement Hub

Solves multi-tenant context issues and creates a clear project lifecycle transition.

User-level auth via MSAL

Actions run under the user's account for accountability and traceability. No service accounts.

All actions logged for traceability

Session logs stored as DevOps attachments. Every input and output flowing through SAGE is recorded.

Three parallel workstreams

SAGE development (Saeed), Axiom hardening (Norik), Engagement Hub replication/enhancement.

X++ code gen as separate workstream

Dev activities may launch a separate MCP-driven experience. 100K+ objects need vectorization.
